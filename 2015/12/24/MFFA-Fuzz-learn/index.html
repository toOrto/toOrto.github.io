MFFA

构造一些可能崩溃但结构正常的媒体文件，然后用合适的软件组件来解码或者是播放这些文件，监控系统，看其是否crash。

系统和设备配置

由于解码部分是通过Stagefright command line接口实现，而普通模拟器并未开启命令行模式，于是可以通过以下几种方式来实现该目的：

  • 重新编译android源代码，在/frameworks/av/cmds/stagefright/目录下修改Android.mk文件，将其中optional换成eng模式。

  • 重新编译android源码，修改device/<vendor>/<target_product>目录下的device.mk文件，即将stagefright模块添加进去。

  • 不编译整个源码，仅编译stagefright模块部分

实现功能（特点）

  1. 分布式发送恶意样本，然后自动log所有的结果。
  2. 解码部分是通过Stagefright command line接口实现的。
  3. crash不重复（检查崩溃PC address是否已存在),并保存tombstone和样本文件
  4. 样本重现（之前就对应log和设备，用于样本重现）

各部分实现

  • 样本生成: 通过AFL生成样本文件，保存到batch目录（略）

  • 样本执行: 包括发送到设备和执行两部分。

    • 分布式发送恶意样本：通过subprocess()子进程实现分布式，实现多个设备同时跑样本。

      对应==>test.py

    • 音频、视频解码：通过Stagefright command line接口实现。分为四种方式，stagefright、stream、sf2和codec四种。这四种方式都是通通过adb push把样本发送到设备上。然后调用各自可执行文件对样本解码。并做log。

      对应==>sp_codec.py、sp_sf2.py、sp_stagefright.py、sp_stream.py

  • 异常监视: 监视是否崩溃。

    • crash监控： 通过adb log记录下所有信息，遍历log找到signal_type（SIGSEGV等），即是可能崩溃的文件

      对应==>get_uniquecrash.py

    • 对应样本文件：遍历log每一行搜索FileName来定位到是哪个样本文件造成的crash。

      对应==>get_uniquecrash.py

    • 保证crash不重复(需要样本重现）：再将crash文件执行一次，通过定位backtrace字符串来检查tombstones里PC address是否已存在，来保证crash不重复

      对应==>get_uniquecrash.py

    • 保证样本重现：即保证在对应的设备上执行对应的样本。通过将log文件和设备对应来实现，从log文件定位出样本文件，放到该log文件对应的设备里重新运行

      对应==>triage.py
